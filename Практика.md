## 🛠 Блок 1: Системное администрирование (Linux 1.2)

### Задание 1.1: Архитектура «Общей папки»

**Задача:** Представь, что тебе нужно создать рабочее пространство для двух разработчиков.

1. Создай группу `web_devs`.
    
2. Создай двух пользователей: `dev1` и `dev2`, добавь их в эту группу.
    
3. Создай папку `/var/www/project`.
    
4. Сделай так, чтобы группа `web_devs` владела этой папкой, а участники могли создавать в ней файлы.
    
5. **Важно:** Настрой так, чтобы любой новый файл в этой папке _автоматически_ наследовал группу `web_devs`.
    

**Ответ:**

Bash

```
# 1-2. Создаем группу и пользователей
sudo groupadd web_devs
sudo useradd -m -G web_devs dev1
sudo useradd -m -G web_devs dev2

# 3-4. Создаем папку и меняем владельца
sudo mkdir -p /var/www/project
sudo chown :web_devs /var/www/project
sudo chmod 770 /var/www/project

# 5. Устанавливаем SetGID бит для наследования группы
sudo chmod g+s /var/www/project
```

## 🤖 Блок 2: Bash-скриптинг и автоматизация (1.3)

### Задание 2.1: Скрипт-монитор «Живой ли сервис?»

**Задача:** Напиши скрипт `check_service.sh`, который:

1. Принимает имя процесса как аргумент (например, `nginx`).
    
2. Проверяет, запущен ли такой процесс.
    
3. Если запущен — пишет "OK". Если нет — пытается запустить его и пишет "Restarting".
    
4. Если аргумент не передан — скрипт должен выдать ошибку и завершиться.
    

**Ответ:**

Bash

```
#!/usr/bin/env bash
set -euo pipefail # Безопасный режим

# Проверка: передан ли аргумент
if [[ -z "${1:-}" ]]; then
    echo "Usage: $0 <service_name>"
    exit 1
fi

SERVICE=$1

# Проверка процесса через pgrep (или ps)
if pgrep "$SERVICE" > /dev/null; then
    echo "$SERVICE is OK"
else
    echo "$SERVICE is down. Restarting..."
    sudo systemctl start "$SERVICE" || echo "Failed to start $SERVICE"
fi
```

### Задание 2.2: Массовая обработка файлов

**Задача:** Используя цикл `for`, напиши однострочник или короткий скрипт, который переименует все файлы `.txt` в текущей папке, добавив к ним префикс `backup_`.

**Ответ:**

Bash

```
for file in *.txt; do
    # Проверяем, что файл существует, чтобы не переименовать саму маску *.txt
    if [[ -f "$file" ]]; then
        mv "$file" "backup_$file"
    fi
done
```

## 🧠 Блок 3: Продвинутые фишки (Атрибуты и функции)

### Задание 3.1: Неубиваемый конфиг

**Задача:** Тебе нужно защитить файл `important.conf` так, чтобы даже `root` не мог его удалить или изменить случайно.

**Ответ:**

Bash

```
# Устанавливаем атрибут immutable (неизменяемый)
sudo chattr +i important.conf

# Проверяем статус
lsattr important.conf
```

### Задание 3.2: Функция-калькулятор

**Задача:** Напиши функцию `calc_sum`, которая принимает два числа и выводит их сумму. Если аргументов меньше двух — выводит ошибку.

**Ответ:**

Bash

```
calc_sum() {
    if [[ $# -lt 2 ]]; then
        echo "Error: Need 2 numbers"
        return 1
    fi
    local result=$(( $1 + $2 ))
    echo "Sum: $result"
}

# Вызов: calc_sum 10 25
```
