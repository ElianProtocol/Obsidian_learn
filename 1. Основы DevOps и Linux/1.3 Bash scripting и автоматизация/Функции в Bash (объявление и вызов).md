Функции в Bash позволяют группировать команды под именем и переиспользовать их с параметрами. Это повышает читаемость скриптов, уменьшает дублирование и упрощает сопровождение.  

## Объявление

В Bash поддерживаются две эквивалентные формы объявления. В контексте переносимости предпочтительнее первая:  

```bash

say_hello() {
  echo "Hello, $1"
}

function say_bye {
  echo "Bye, $1"
}
```

Имя функции следует правилам именования идентификаторов. Тело функции выполняется в текущем процессе оболочки, если не запускается в подпроцессе через скобки.  

## Вызов и аргументы

Вызов функции аналогичен вызову команды:  

```nginx

say_hello "DevOps"
```

Внутри функции доступны:  

- 1,1,2, … — позиционные параметры;
- $# — число аргументов;
- "$@" — все аргументы как отдельные слова;
- "$*" — все аргументы слиты в одну строку.

Рекомендуется применять цитирование, чтобы сохранить границы аргументов. Для перебора удобно использовать shift:  

```bash

print_args() {
  i=1
  while [ "$#" -gt 0 ]; do
    printf "%d: %s\n" "$i" "$1"
    i=$((i+1))
    shift
  done
}
```

Сравнение "@"и"@"и"*":  

|Конструкция|Семантика при кавычках|Пример аргументов|Результат итерации|
|---|---|---|---|
|"$@"|Список слов|["a b", "c"]|"a b", "c"|
|"$*"|Одна строка|["a b", "c"]|"a b c"|

## Возврат значений

Функции возвращают код завершения (0–255) через return. Для передачи данных используйте вывод и подстановку команд:  

```bash
sum() {
  local a=$1 b=$2
  echo $((a + b))
}

x=$(sum 2 3)   # x=5
check() { [ -f "$1" ]; }  # код завершения: 0 если файл существует
check "/etc/hosts" || echo "нет файла"
```

Код завершения последней команды в функции равен $? и используется оболочкой при проверках с && и ||.  

## Область видимости и побочные эффекты

По умолчанию переменные глобальны для скрипта. Используйте local для ограничения видимости:  

```bash
counter=0
inc() {
  local step=${1:-1}
  counter=$((counter + step))
}
```

local влияет лишь на переменные, но не изолирует окружение: изменение текущего каталога, файлов и дескрипторов влияет на внешний контекст. Для изоляции используйте подпроцесс:  

```bash

tmp=$( ( cd /; pwd ) )  # текущий каталог наружу не меняется
```

Начиная с Bash 4.3 доступен nameref:  

```perl

set_value() {
  local -n ref=$1
  ref=$2
}
set_value var_name new_value
```

## Экспорт и переопределение

Функцию можно экспортировать дочерним оболочкам:  

```javascript

myfn() { echo "child"; }
export -f myfn
bash -c 'myfn'
```

Для защиты от модификации используйте readonly -f имя. Список функций: declare -F, показать тело: declare -f имя.  

Функции могут переопределять встроенные команды. Для вызова оригинала используйте builtin:  

```bash

cd() {
  echo "Переход в $1"
  builtin cd "$1"
}
```
