
Циклы в Bash позволяют повторять действия над списками значений, до/пока выполняется условие или по счётчику. Они управляют потоком выполнения, опираясь на код возврата команд (0 — истина, иначе — ложь), а не на булевы выражения в привычном понимании.  

## Цикл for

Вариант по списку использует разбиение слов и подстановки оболочки:  

```bash

for item in alpha beta gamma; do
  echo "Элемент: $item"
done
```

Часто список формируется подстановками:  

```bash

for f in *.log; do
  gzip -9 "$f"
done
```

- Важно экранировать переменные: "$f" предотвращает разбиение по пробелам.
- Для пустых совпадений *.log можно включить опцию nullglob: shopt -s nullglob.

Перебор массива:  

```bash

arr=(dev stage prod)
for env in "${arr[@]}"; do
  echo "$env"
done
```

C-подобный вариант удобен для счётчиков:  

```bash

for ((i=0; i<5; i++)); do
  printf "%d " "$i"
done
```

Внутри (( )) сравнения и инкременты — арифметические, без $.  

## Циклы while и until

while выполняется, пока команда-условие возвращает 0:  

```bash

count=0
while (( count < 3 )); do
  echo "Попытка $count"
  ((count++))
done
```

until выполняется, пока команда-условие возвращает НЕ 0 (то есть «пока не наступило истинное состояние»):  

```bash

until curl -fsS http://localhost:8080/health >/dev/null; do
  echo "Ждём сервис..."
  sleep 1
done
```

Такой паттерн используют для ожидания готовности зависимостей.  

## Чтение потоков и файлов

Правильный способ построчного чтения — через read -r и перенаправление, чтобы избежать потери переменных из-за пайплайна в подпроцессе:  

```bash

while IFS= read -r line; do
  printf ">> %s\n" "$line"
done < /etc/hosts
```

- IFS= сохраняет ведущие/концевые пробелы.
- -r запрещает интерпретацию обратной косой черты.

Если нужно фильтровать поток, используйте процессную подстановку:  

```bash

while IFS=, read -r user role; do
  echo "User=$user Role=$role"
done < <(tail -n +2 users.csv)
```

## Управление потоком: break и continue

Прерывание текущей итерации:  

```bash

for x in {1..10}; do
  (( x % 2 == 0 )) && continue
  echo "Нечётное: $x"
done
```

Выход из одного или нескольких уровней:  

```bash

for a in A B; do
  for b in 1 2 3; do
    [[ $b -eq 2 ]] && break 2
    echo "$a$b"
  done
done
```

## Сопоставление конструкций

|Конструкция|Условие|Типичная область|Особенности|
|---|---|---|---|
|for items|нет (список значений)|Перебор файлов, массивов, аргументов|Чувствителен к word splitting и glob; цитируйте переменные|
|for ((…))|арифметика|Счётчики, ограниченные итерации|Быстро, компактно, без $ внутри (( ))|
|while|true при коде 0|Чтение потоков, поллинг, ожидания|Условие — команда; удобно с test, [[ ]], (( ))|
|until|true при коде ≠ 0|Ожидание «пока не станет доступно»|Инверсия while, читаемо для «ждать до готовности»|

## Практические нюансы

- Используйте [[ … ]] для безопасных строковых проверок и (( … )) для чисел.
- В сценариях ожидания добавляйте таймаут:

```bash

for ((i=0; i<30; i++)); do
  curl -fsS http://svc/health && break
  sleep 1
done
```

- Для больших наборов файлов предпочтительнее find -print0 + while read -d '' — корректно обрабатывает имена с пробелами и переносами.
