Bash — это командный интерпретатор и язык сценариев, широко используемый для автоматизации в системах семейства Unix. Его сильные стороны — интерактивность, богатая система подстановок, удобные средства работы с процессами и потоками ввода-вывода.  

## Сценарий Bash: минимальная структура

Скрипт — это текстовый файл с командами, которые Bash выполняет последовательно.  

```d

#!/usr/bin/env bash
set -Eeuo pipefail
# Комментарии начинаются с # и игнорируются интерпретатором

echo "Hello, Bash!"
```

- Shebang сообщает системе, какой интерпретатор использовать.
- Опции часто включают:
- -e: завершать при ошибке команды;
- -u: ошибка при обращении к несуществующей переменной;
- -o pipefail: ошибка конвейера, если сбой в любой команде;
- -E: сохранять обработчики ERR в функциях и подшеллах.

## Переменные и кавычки

В Bash переменные не типизированы и создаются присваиванием. Пробелы вокруг = недопустимы.  

```bash

name="Alice"
count=3
greeting=Hello        # без кавычек — слова разделяются пробелами при подстановке
echo "$greeting, $name. Count=$count"
```

Кавычки определяют, как интерпретируются пробелы, метасимволы и подстановки.  

|Форма|Эффект|Пример|Результат|
|---|---|---|---|
|без кавычек|словоразделение, glob-расширение|echo $name/*.log|подставит список файлов|
|"двойные"|сохраняют пробелы, разрешают подстановки|echo "namename((1+2))"|Alice 3|
|'одинарные'|буквально всё внутри|echo 'namename((1+2))'|namename((1+2))|

## Расширения и подстановки

- Подстановка переменных: ${var}, с модификаторами.
- Командная подстановка: $(cmd) возвращает вывод команды.
- Арифметика: $((expr)) вычисляет выражение.
- Шаблоны (globbing): *, ?, [abc] для соответствия именам файлов.
- Скобочные расширения: file_{a,b}.txt → file_a.txt file_b.txt.

```bash

files=$(ls *.log 2>/dev/null || true)
echo "Найдено: ${#files}"
num=$((10 + 5))
echo "sum=$num"
```

Полезные формы параметрического расширения:  

- ${var:-default} — значение по умолчанию;
- ${var:?msg} — ошибка, если var пусто;
- ${var%pattern}, ${var%%(%%)pattern} — обрезка по шаблону справа.

## Управляющие операторы и статус выхода

Каждая команда возвращает код завершения ($?): 0 — успех, иное — ошибка. Это основа условных связок.  

- ; — последовательное выполнение;
- && — выполнить следующую, если успех;
- || — выполнить следующую, если ошибка;
- | — конвейер: передача stdout следующей команде.

```bash

build && echo "OK" || echo "FAIL"
grep pattern file.txt | wc -l
echo "code=$?"
```

[[ … ]] — расширенный тест Bash (безопаснее для строк и шаблонов), в отличие от [ … ].  

```bash

if [[ -f "$path" && "$name" == foo* ]]; then
  echo "match"
fi
```

## Потоки и перенаправления

Стандартные потоки: stdin (0), stdout (1), stderr (2). Перенаправления управляют ими:  

- > перезаписать, >> дописать;
- 2> перенаправить ошибки;
- 2>&1 объединить stderr со stdout.

```lua

cmd >out.log 2>err.log
cmd >>all.log 2>&1
grep -i error <input.txt | sort | uniq -c
```

В конвейерах учитывайте -o pipefail, чтобы ловить ошибки промежуточных команд.  

## Встроенные и внешние команды, совместимость

Bash имеет встроенные команды (cd, echo, declare, mapfile). Определить природу команды помогает:  

```bash

type cd
type ls
```

Bash расширяет POSIX sh: массивы, ассоциативные массивы, [[ ]], { } группы, арифметика, скобочные расширения. Для максимальной переносимости ограничиваются sh-конструкциями; для полноценной автоматизации в DevOps чаще используют богатые возможности Bash.
