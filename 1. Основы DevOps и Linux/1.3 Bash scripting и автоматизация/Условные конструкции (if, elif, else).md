Условные конструкции в Bash позволяют выполнять команды в зависимости от результата проверки. Ключевой принцип: любая команда в Unix возвращает код завершения; 0 означает «успех», ненулевое значение — «ошибка». Конструкция if оценивает именно код завершения команды или выражения.  

## Синтаксис и формы записи

Базовые варианты:  

```bash

if команда; then
  ...
fi

if команда
then
  ...
fi

if команда; then
  ...
elif другая_команда; then
  ...
else
  ...
fi
```

Ключевое слово then может стоять на новой строке или после точки с запятой. Ветвление elif используется для последовательной проверки нескольких условий, else — для альтернативной ветки.  

## Тесты: строки, числа, файлы

Обычно условие задаётся через [ … ], [[ … ]] или (( … )). Часто применяются следующие проверки:  

|Проверка|Значение|
|---|---|
|-z "$s"|Пустая строка|
|-n "$s"|Непустая строка|
|"a"="a"="b"|Равенство строк|
|"a"!="a"!="b"|Неравенство строк|
|-e path|Существование файла/каталога|
|-f path|Обычный файл|
|-d path|Каталог|
|-s path|Файл не пуст|
|-r/-w/-x path|Права: чтение/запись/исполнение|
|-nt / -ot|«Новее» / «старее»|
|-eq/-ne/-lt/-le/-gt/-ge|Числовые сравнения|

Примеры:  

```nginx

if [ -f "$cfg" ] && [ -s "$cfg" ]; then echo "Конфиг найден"; fi
if [ -z "$USER" ]; then echo "Нет переменной USER"; fi
if [ "$a" -gt 10 ]; then echo "a > 10"; fi
```

## [ … ] vs [[ … ]] vs (( … ))

- [ … ] — POSIX-совместимый тест. Требует тщательного экранирования и кавычек вокруг переменных, чувствителен к пробелам и подстановкам оболочки.
- [[ … ]] — расширенный тест Bash. Не выполняет разделение по пробелам и тильде, безопаснее с незаданными переменными, поддерживает шаблоны и регулярные выражения.
- (( … )) — арифметическая оценка для целых чисел.

Примеры:  

```nginx

# Расширенные проверки
if [[ "$env" == prod* ]]; then echo "Продакшен"; fi
if [[ "$tag" =~ ^v[0-9]+\.[0-9]+$ ]]; then echo "Семантическая версия"; fi

# Арифметика
retries=3
if (( retries > 2 )); then echo "Слишком много попыток"; fi
```

## Комбинирование условий и возврат статусов

Внутри [[ … ]] можно использовать && и ||:  

```bash

if [[ -d "$dir" && -w "$dir" ]]; then
  echo "Каталог доступен для записи"
elif [[ -e "$dir" ]]; then
  echo "Путь существует, но не каталог или нет прав"
else
  echo "Путь отсутствует"
fi
```

Также часто применяют саму команду как условие:  

```bash

if grep -q "enabled" "$cfg"; then
  echo "Флаг включен"
else
  echo "Флаг выключен"
fi
```

Если команда завершается успешно (код 0), выполняется then; иначе — elif/else.  

## Типичные ошибки и рекомендации

- В [ … ] обязательны пробелы вокруг операторов: [ "a"="a"="b" ], а не ["a"="a"="b"].
- Всегда заключайте переменные в кавычки в [ … ]: [ -z "$var" ]. В [[ … ]] это менее критично, но желательно.
- Для шаблонов и регулярок используйте [[ … ]] с операторами == и =~.
- Для чисел предпочтительнее (( … )), это короче и безопаснее.
- При возможном отсутствии переменных используйте подстановку по умолчанию: [[ -z ${VAR:-} ]].
