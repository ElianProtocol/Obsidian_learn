## Ключевые понятия

В Bash различают два класса переменных:  

- Локальные (shell variables) — существуют только в текущем процессе оболочки.
- Переменные среды (environment variables) — экспортированные значения, доступные дочерним процессам.

Локальную переменную создают простым присваиванием:  

```ini

APP_DIR=/opt/app
```

Переменную среды создают экспортом:  

```bash

export APP_ENV=prod
# или одним выражением
export PATH="$PATH:/opt/app/bin"
```

Часто встречающиеся переменные среды: PATH, HOME, USER, SHELL, LANG, PWD, IFS. Они влияют на поведение программ и оболочки.  

## Область видимости и наследование

- Локальные переменные видны только текущему процессу Bash.
- Переменные среды наследуются дочерними процессами (скриптами, утилитами, подзапущенными оболочками).

Пример различий:  

```julia

FOO=local
export BAR=env

bash -c 'echo "$FOO | $BAR"'
# вывод:  | env
```

FOO не виден дочерней оболочке, BAR — виден.  

Одноразовое переопределение только для команды:  

```ini

APP_ENV=staging ./deploy.sh
```

В этом случае окружение изменено лишь для запускаемого процесса.  

Полная «чистая» среда:  

```bash

env -i PATH=/usr/bin:/bin bash -c 'env'
```

## Команды для работы

- Просмотр:
- `set` — все переменные и функции оболочки.
- `env` или `printenv` — переменные среды.
- `declare -x` — только экспортированные.
- Экспорт/снятие экспорта:
- `export VAR` — сделать переменную переменной среды.
- `export VAR=value` — создать и экспортировать сразу.
- `declare -x VAR=value` — эквивалентно export.
- Удаление:
- `unset VAR` — убрать переменную (и из среды, и из локальных).
- Защита от изменения:

```  
readonly VERSION=1.2.3  

# readonly PATH — запретить модификацию PATH в текущей сессии

```  

## Подстановка и кавычки

Кавычки влияют на подстановку и интерпретацию пробелов:  

```bash

NAME="my app"
echo $NAME      # word splitting -> два слова
echo "$NAME"    # одно слово "my app"
```

Безопасные практики: заключать переменные в двойные кавычки, особенно при путях, шаблонах и данных из внешних источников.  

Полезные формы параметрической подстановки:  

```bash

echo "${REGION:-us-east-1}"   # значение по умолчанию
: "${REQUIRED?Переменная REQUIRED не установлена}"  # остановить, если нет
```

## Сравнение

|Характеристика|Локальная переменная|Переменная среды|
|---|---|---|
|Область видимости|Текущий Bash|Текущий и дочерние процессы|
|Наследование|Нет|Да|
|Создание|VAR=value|export VAR[=value]|
|Просмотр|set, declare|env, printenv, declare -x|
|Удаление|unset VAR|unset VAR|

## Чувствительность к регистру и типы

Имена переменных чувствительны к регистру: VAR и var — разные. По традиции переменные среды пишутся заглавными буквами (PATH, HOME), локальные — любыми.  

В среду экспортируются только строки. Массивы и ассоциативные массивы не могут быть корректно экспортированы:  

```makefile

arr=(a b c)
export arr           # будет преобразовано в строку; не используйте так
```

## Нюансы и конфигурационные файлы

Инициализация окружения зависит от типа сессии:  

- Логин-оболочка читает системные и пользовательские файлы профиля.
- Нелогин-оболочка обычно читает интерактивные настройки.
- Скрипты запускаются в неинтерактивной оболочке; многие интерактивные файлы не подключаются автоматически.

Расширение PATH выполняют аккуратно:  

```bash

PATH="/opt/mytool/bin:$PATH"
export PATH
```

Секреты в переменных среды не выводят и не логируют; предпочитают механизм передачи в процесс без сохранения в истории оболочки.
