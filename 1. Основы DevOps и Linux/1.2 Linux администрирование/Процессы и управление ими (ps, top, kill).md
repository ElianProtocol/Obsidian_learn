
## Модель процессов в Linux

Процесс — это исполняемая программа с собственным адресным пространством и контекстом. Каждый процесс имеет:  

- PID — уникальный идентификатор;
- PPID — идентификатор родительского процесса;
- UID/GID — владельца и группу;
- Приоритет и «вежливость» (nice), влияющие на планирование;
- Состояние (STAT), отражающее текущую активность.

Типовые состояния в поле STAT:  

- R — выполняется на CPU;
- S — спит (ожидает событие) и может быть прерван;
- D — непрерываемый сон (обычно ожидание I/O);
- T — остановлен (например, сигналом SIGSTOP);
- Z — «зомби» (завершён, но не собран родителем).

Дополнительные флаги: N (пониженный приоритет), < (высокий приоритет), s (лидер сессии), + (на переднем плане).  

## ps: «снимок» состояния

Команда ps выводит мгновенный снимок процессов. Распространённые варианты:  

```dos

ps aux
ps -ef
```

Первый стиль — BSD-формат, второй — System V. Для точной выборки используйте фильтры и форматирование:  

```perl

# Процессы текущего пользователя, отсортированы по CPU
ps -u "$USER" -o pid,ppid,%cpu,%mem,stat,ni,cmd --sort=-%cpu | head

# По имени команды
ps -C nginx -o pid,ppid,stat,%cpu,%mem,cmd

# По PID
ps -p 1234 -o pid,ppid,stat,etime,%cpu,%mem,cmd
```

Сортировка возможна по любому числовому полю через --sort, добавляя «-» для убывания. Поле STAT быстро показывает «проблемные» процессы: D трудно завершить немедленно, Z указывает на необходимость работы с родителем.  

Для сценариев, где нужен список без «лишнего», удобно явно перечислять поля через -o.  

## top: интерактивный мониторинг

top отображает процессы в реальном времени, обновляя статистику. Полезные клавиши:  

- P/M/T — сортировка по CPU/памяти/времени;
- k — отправить сигнал процессу по PID;
- r — изменить nice;
- 1 — показать загрузку по каждому CPU;
- H — отобразить потоки;
- c — переключить командную строку/имя процесса;
- f/F — выбор и порядок столбцов.

Пример для неинтерактивного режима (удобно в автоматизации):  

```bash

top -b -n1 | head -50
```

Значение %CPU в top усредняется между обновлениями; короткоживущие процессы могут «не попасть» в выборку, где ps сработает лучше при частом опросе.  

## Сигналы и kill

Сигналы — механизм управления процессами: завершение, пауза, продолжение, перезапуск. Команда kill отправляет сигнал по PID. По умолчанию посылается SIGTERM (15) — корректное завершение.  

```bash

# Список сигналов
kill -l

# Корректно завершить
kill -TERM 1234

# Принудительно (последний способ)
kill -KILL 1234

# Остановить и продолжить
kill -STOP 1234
kill -CONT 1234
```

Часто используемые сигналы:  

|Сигнал|Номер|Назначение|
|---|---|---|
|SIGHUP|1|Перечитать конфигурацию/перезапуск|
|SIGINT|2|Прерывание с терминала|
|SIGTERM|15|Корректное завершение|
|SIGKILL|9|Немедленное завершение|
|SIGSTOP|19|Остановка выполнения|
|SIGCONT|18|Возобновление|

Важно учитывать права: отправлять сигналы чужим процессам может только root или владелец. Состояние D чаще всего не реагирует на сигналы до завершения операции I/O. Зомби-процессы не «убиваются»: необходимо дождаться wait() со стороны родителя или завершить родительский процесс.  

Комбинируя инструменты:  

```perl

# Найти «тяжёлые» процессы и завершить их корректно
ps -eo pid,%cpu,%mem,stat,cmd --sort=-%cpu | head
kill -TERM <PID>
```
