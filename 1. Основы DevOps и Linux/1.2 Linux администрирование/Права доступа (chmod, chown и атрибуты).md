
Модель прав доступа в Linux опирается на владельца объекта, его группу и остальных пользователей, а также на три базовых бита: чтение (r), запись (w) и исполнение (x). Эта система определяет, кто и какие операции может выполнять с файлами и каталогами, и является фундаментом безопасности многопользовательских систем.  

## Модель прав: r, w, x и классы пользователей

Правильное чтение строки прав в выводе ls -l помогает быстро оценить настройки объекта:  

```kotlin

$ ls -l /var/www/index.html
-rw-r----- 1 www-data www-data 1024 Oct 10 10:00 index.html
```

Первый символ задает тип (например, - файл, d каталог). Далее идут три тройки: для владельца, группы и остальных.  

Значения битов различаются для файлов и каталогов:  

|Бит|Файлы|Каталоги|
|---|---|---|
|r|Чтение содержимого|Просмотр списка имен (ls)|
|w|Изменение содержимого|Создание/удаление/переименование объектов|
|x|Исполнение как программы|Вход в каталог (cd) и доступ к именам|

Числовое кодирование: r=4, w=2, x=1. Пример: 640 = rw- r-- ---; 755 = rwx r-x r-x.  

## chmod: символический и числовой режимы

chmod изменяет биты доступа. Два подхода — числовой и символический.  

- Числовой:

```bash

chmod 640 index.html
chmod 755 /usr/local/bin/tool
```

- Символический с классами u (owner), g (group), o (others), a (all):

```css

chmod u+x script.sh
chmod g-w,o-rwx secret.txt
chmod -R g+w /srv/projects
chmod -R a-X /var/share
```

Опция -R действует рекурсивно. Специальный флаг X добавляет x только каталогам и уже исполняемым файлам.  

Специальные биты:  

- setuid (4xxx): процесс наследует EUID владельца файла.
- setgid (2xxx): процесс наследует EGID; для каталогов новые файлы наследуют группу каталога.
- sticky (1xxx): для каталогов удалять/перемещать файлы может только владелец файла, владелец каталога или root.

Примеры:  

```bash

chmod 4755 /usr/local/bin/helper      # setuid
chmod g+s /var/www                    # setgid на каталог
chmod +t /tmp                         # sticky bit на /tmp
```

## chown и chgrp: владелец и группа

chown меняет владельца и/или группу. Использовать можно как root или с соответствующими привилегиями.  

```bash

chown alice report.csv
chown :devops report.csv
chown alice:devops report.csv
chown -R www-data:www-data /var/www
```

Для изменения только группы можно применять chgrp:  

```bash

chgrp -R devops /srv/repos
```

## Атрибуты файловой системы: chattr/lsattr

Помимо POSIX-прав некоторые файловые системы (например, ext4) поддерживают неизменяемые и другие флаги.  

Просмотр:  

```applescript

lsattr file.txt
```

Ключевые флаги:  

- i: immutable — запрещены изменение, удаление, линковка.
- a: append-only — разрешено только дозаписывать.

Управление:  

```css

chattr +i critical.conf
chattr +a audit.log
chattr -i critical.conf
```

## umask и права по умолчанию

Права новых объектов формируются из базовых значений за вычетом umask. Для файлов базовое 666, для каталогов 777.  

Примеры:  

```bash

umask            # показать текущую маску, например 022
# Файлы: 666-022=644, каталоги: 777-022=755

umask 027         # новые файлы 640, каталоги 750
```
