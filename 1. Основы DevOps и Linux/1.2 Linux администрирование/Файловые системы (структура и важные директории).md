Файловая система в Linux описывает способ хранения и организации данных на носителях и определяет иерархию каталогов, видимую пользователю. Стандарт Filesystem Hierarchy Standard (FHS) задает общие правила размещения системных и пользовательских данных, что упрощает администрирование и переносимость. Важно понимать различие между реальными и виртуальными файловыми системами: первые хранят данные на диске (ext4, XFS, Btrfs), вторые предоставляют интерфейсы ядра и устройств в виде файлов (/proc, /sys, /dev).  

## Ключевые понятия

- inode — структура, содержащая метаданные файла (владельцы, права, метки времени, указатели на блоки). Имя файла хранится в каталоге и ссылается на inode.
- Жесткая и символическая ссылки — жесткая ссылка указывает на тот же inode, символическая (symlink) — на путь; удаление исходного файла не затрагивает жесткую ссылку, но делает «битой» символическую.
- Точка монтирования — каталог, в который встраивается другая файловая система. Один каталог — одна смонтированная ФС.
- Виртуальные ФС — представляют объекты ядра и устройств как файлы: /proc (процессы, параметры ядра), /sys (sysfs: устройства, драйверы), /dev (устройства).

## Иерархия и назначение каталогов

|Каталог|Назначение|Примечание|
|---|---|---|
|/|Корень иерархии|Начало всех путей|
|/boot|Ядро, initramfs, загрузчик|Отдельный раздел в продакшене встречается часто|
|/etc|Конфигурации системы|Текстовые файлы, версионируются|
|/bin, /sbin|Базовые утилиты и системные бинари|Доступны в ранней загрузке|
|/usr|Большинство пользовательских приложений|/usr/bin, /usr/lib, /usr/share|
|/usr/local|Локально установленные программы|Не затрагиваются пакетным менеджером|
|/var|Данные, меняющиеся во времени|/var/log, /var/lib, /var/spool|
|/home|Домашние каталоги пользователей|Пользовательские данные и конфиги|
|/root|Домашний каталог root|Изолирован от /home|
|/tmp|Временные файлы|Может очищаться при перезагрузке|
|/opt|Сторонние приложения|Часто self-contained пакеты|
|/dev|Файлы устройств|Создаются udev|
|/proc|Состояние ядра и процессов|Псевдофайлы, не занимают диск|
|/sys|Дерево устройств ядра|Настройка через атрибуты|
|/run|Краткоживущие runtime-данные|Сокеты, PID-файлы|
|/mnt, /media|Точки временного и авто-монтажа|/mnt — админский, /media — авто-медиа|
|/srv|Данные сервисов|Веб-контент, репозитории|

Практически полезно выделять отдельные разделы для /var и /home для управления ростом логов и пользовательских данных. Для /tmp и контейнерных узлов уместны монтирования с ограничениями безопасности (noexec, nodev, nosuid).  

## Команды и конфигурация

Просмотр устройств и монтирований:  

```bash

lsblk -f
mount | column -t
findmnt -t ext4,xfs
df -hT
du -sh /var/log/* | sort -h
```

Пример записи в /etc/fstab для отдельного раздела под логи и безопасного /tmp:  

```bash

UUID=xxxx-xxxx  /var/log  xfs   noatime,nodev,nosuid        0 2
tmpfs           /tmp      tmpfs noexec,nodev,nosuid,mode=1777 0 0
```

Монтирование без перезагрузки и проверка:  

```bash

sudo mount -a
findmnt /var/log
```

Создание ссылок и работа с inode:  

```bash

ln file.txt hardlink.txt
ln -s /etc/nginx/nginx.conf nginx.conf.link
stat file.txt hardlink.txt
ls -li file.txt hardlink.txt nginx.conf.link
```

К виртуальным каталогам следует относиться как к интерфейсу ядра: чтение /proc/cpuinfo или изменение параметров через /proc/sys и /sys влияет на систему немедленно, но не «пишет» на диск.
