Учетные записи определяют, кто может входить в систему и какие действия выполнять. Каждый пользователь имеет уникальный UID, принадлежит основной группе (GID) и может состоять в дополнительных группах. Системные учетные записи (обычно UID < 1000) обслуживают службы и не предназначены для интерактивного входа.  

Основные файлы:  

- /etc/passwd — список пользователей (имя, UID, GID, shell, домашний каталог)
- /etc/shadow — пароли и политика их изменения (доступ только root)
- /etc/group — группы и их состав
- /etc/skel — шаблон содержимого для новых домашних каталогов

Пример строки в /etc/passwd:  

```ruby

alice:x:1001:1001:Alice User:/home/alice:/bin/bash
```

## Создание и удаление пользователей

Базовые команды:  

- useradd — низкоуровневое создание учетной записи
- adduser — диалоговая обертка (в некоторых дистрибутивах отсутствует)
- userdel — удаление
- groupadd — создание группы

Примеры:  

```bash

# создать пользователя с домашним каталогом и оболочкой bash
useradd -m -s /bin/bash alice

# задать пароль
passwd alice

# создать пользователя и добавить в дополнительные группы (например, docker, wheel)
useradd -m -s /bin/bash -G docker,wheel devops

# создать системную учетную запись без входа в систему
useradd -r -M -s /usr/sbin/nologin prometheus

# удалить пользователя и его домашний каталог
userdel -r alice
```

Полезные параметры useradd:  

- -m — создать домашний каталог из /etc/skel
- -s — логин-оболочка
- -u — явный UID
- -g — основная группа (или имя, или GID)
- -G — дополнительные группы
- -r — системная учетная запись

## Модификация, блокировка и политика паролей

Изменение атрибутов:  

```bash

# сменить оболочку
usermod -s /bin/zsh devops

# добавить/удалить из группы
usermod -aG docker devops
gpasswd -d devops docker
```

Блокировка входа:  

```shell

# временно заблокировать пароль (префикс '!' в /etc/shadow)
passwd -l devops

# разблокировать
passwd -u devops
```

Политика паролей и срок действия:  

```nginx

# задать максимальный срок 90 дней, предупреждать за 7 дней
chage -M 90 -W 7 devops

# просмотреть текущие значения
chage -l devops
```

## Группы и sudo

Основная группа влияет на новые файлы по умолчанию, дополнительные расширяют доступ. Проверка:  

```bash

id devops
groups devops
```

Доступ через sudo настраивается в /etc/sudoers (редактировать через visudo для валидации синтаксиса):  

```sql

# дать группе wheel право выполнять любые команды с паролем
%wheel ALL=(ALL) ALL
```

## Права на файлы и специальные биты

Модель прав: владелец, группа, остальные; биты r (читать), w (записывать), x (выполнять). Числовая запись суммирует биты: r=4, w=2, x=1.  

Примеры:  

```bash

# права 750: владелец rwx, группа r-x, остальные ---
chmod 750 /srv/app

# смена владельца и группы
chown devops:docker /srv/app

# только владелец может удалять в каталоге, даже если есть права записи (sticky)
chmod +t /tmp

# setgid на каталоге: новые файлы наследуют группу каталога
chmod g+s /srv/shared

# setuid на исполняемом файле (исполняется с правами владельца)
chmod u+s /usr/local/bin/tool
```

umask задает маску запрета прав для новых файлов:  

```bash

# временно
umask 027

# проверить, какие права получит новый файл/каталог
umask
```

Сопоставление символов и чисел:  

- rwx = 7, rw- = 6, r-x = 5, r-- = 4, -wx = 3, -w- = 2, --x = 1, --- = 0

При работе с правами следует учитывать минимально необходимый доступ и избегать широких масок вроде 777 на каталоги приложений.
