Фундамент работы в терминале. Без них невозможно перемещаться по системе и управлять файлами.

| **Команда**  | **Полное название** | **Что делает**                                                                            | **Пример использования** |
| ------------ | ------------------- | ----------------------------------------------------------------------------------------- | ------------------------ |
| **`cd`**     | Change Directory    | Переход в указанную директорию.                                                           | `cd /var/log`            |
| **`mkdir`**  | Make Directory      | Создание новой пустой папки.                                                              | `mkdir my_project`       |
| **`rm`**     | Remove              | Удаление файлов.                                                                          | `rm file.txt`            |
| **`rm -rf`** | Recursive Force     | **Опасная команда!** Рекурсивное и принудительное удаление директории со всем содержимым. | `rm -rf old_dir`         |
| **`touch`**  | -                   | Создание пустого файла или обновление времени изменения существующего.                    | `touch config.yaml`      |
| **`chmod`**  | Change Mode         | Изменение прав доступа к файлу/папке (кто может читать, писать, запускать).               | `chmod +x script.sh`     |

**Advanced Topics**

<mark style="background: #D2B3FFA6;">Навигация и работа с файлами:</mark>
	find: Поиск файлов по имени
		Пример: find /var/log -name "*.log" - найти все лог-файлы в директории /var/log
	locate: мгновенный поиск по индексированной базе.
		Пример: locate nginx.conf
	tree: дерево директорий.
		Пример: tree -L 2 - показать структуру папок на два уровня вглубь.
	ln: создание ссылок.
		Пример: ln -s /path/to/file linkname - создать символическую ссылку
		ln s /path/to/file linkname - создать жесткую ссылку
	stat: детальная информация (права, inode, время доступа).
		Пример: stat config.yaml
	file: определение типа содержимого.
		Пример: file script.sh - покажет, является ли файл bash-скриптом или обычным текстом

<mark style="background: #D2B3FFA6;">Потоки и перенаправления: </mark>
Настройка среды выполнения программ.
	env: список всех активных переменных.
	export: создание переменной, доступной дочерним процессам.
		Пример: export DB_USER="admin"
	PATH: важнейшая переменная со списком путей к исполняемым файлам.
	Файлы конфигурации:
		.bashrc: настройки для интерактивных сессий (каждый новый терминал).
		.bash_profile / .profile: настройки при входе в систему (login).

<mark style="background: #D2B3FFA6;">Работа с текстом (Анализ логов):</mark>
Инструментарий для поиска и трансформации данных.
	grep: поиск по шаблону.
		Пример: grep -i "critical" app.log (регистронезависимый поиск).
	awk: обработка колонок
		Пример: awk '(print $1)' access.log - вывести только IP-адреса (первую колонку)
	sed: замена текста
		Пример: sed -i 's/old/new/g' config.cfg
	sort & uniq: сортировка и поиск уникальных строк.
		Пример: cat ips.txt | sort | uniq -c -посчитать количество уникальных IP-адресов
	wc: подсчет строк/слов:
		Пример: wc -l file.txt - узнать количество строк
	xargs: выполнение команды для каждой строки из ввода.
		Пример: find . -name "*.tmp" | xargs rm - найти и удалить все временные файлы.

<mark style="background: #D2B3FFA6;">SSH: Удаленное управление</mark>
Инструменты для безопасного доступа к серверам.
	ssh-keygen: создание пары ключей (публичный и приватный)
	ssh-copy-id: перенос вашего ключа на сервер для входа без пароля.
		Пример: ssh-copy-id user@server-ip
	~/.ssh/config: файл для создания алиасов (коротких имен) серверов.
	scp / rsync: копирование файлов
		Пример: rsync -avz local_folder/ user@server:/remote_folder/ (быстрая синхронизация)
