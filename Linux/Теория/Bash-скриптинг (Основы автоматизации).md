Скрипт - это просто текстовый файл, содержащий последовательность команды, которые ты обычно вводишь вручную.

<mark style="background: #FFB86CA6;">1.Анатомия скрипта</mark>
Любой скрипт должен начинаться с Shebang(шебанг). Это путь к интерпретатору, который будет выполнять код.
	Строка #!/bin/bash
	Права доступа: Чтобы скрипт можно было запустить, ему нужно дать права на выполнение:
		chmod +x myscript.sh

<mark style="background: #FFB86CA6;">2.Переменные в скриптах</mark>
В отличие от командной строки, в скриптах мы часто храним промежуточные данные.
```
#!/bin/bash
NAME="DevOps_Student" %% Присваивание (без пробелов!) %%
echo "Привет, $NAME" %% Использование переменной через $ %%
```
<mark style="background: #FFB86CA6;">3.Управляющие конструкции (Логика)</mark>
Условия (if/else)
Используются для проверки состояния системы(например, существует ли файл или запущен ли процесс).
```
if [ -f "/etc/nginx/nginx.conf" ]; then
	echo "Конфиг nginx найден"
else
	echo "Ошибка: конфиг отсутствует"
	exit 1 %% Завершение с кодом ошибки %%
fi
```
Циклы (for/while)
Используются для массовых операций (например, переименование 100 файлов или опрос 10 серверов).

```
%% Цикл по списку файлов %%
for file in *.log; do
	echo "Обработка файла: $file"
	tar -czf "$file.tar.gz" "$file"
done
```
<mark style="background: #FFB86CA6;">4.Коды завершения (Exit Codes)</mark>
Каждая команда после выполнения возвращает число от 0 до 255.
	0 - успех
	1 и выше - ошибка
	Как проверить: Посмотри переменную $? сразу после выполнения команды.
<mark style="background: #FFB86CA6;">5.Функции</mark>
Позволяют группировать код для повторного использования.
```
print_status() {
	echo "--- [STATUS]: $1 ---"
}
print_status "Запуск бэкапа"
```
