## Git Commit 01

### Цель

Узнайте, как работать с ветками Git.
### Инструкции

1. Найдите репозиторий Git (или создайте новый), содержащий хотя бы один коммит.
2. Создайте новую ветку с названием "dev".
3. Измените один из файлов в репозитории.
4. Создать новый коммит
5. Убедитесь, что созданный вами коммит находится только в ветке "dev".

### После выполнения упражнения

Ответьте на следующие вопросы:

1. Почему ветвления полезны? Приведите пример одной реальной ситуации использования ветвей.


Решение:
## Ветка 01 - Решение

```
cd some_repository
echo "master branch" > file1
git add file1
git commit -a -m "added file1"
git checkout -b dev
echo "dev branch" > file2
git add file2
git commit -a -m "added file2"
```

Проверка:

```
git log (you should see two commits)
git checkout master
git log (you should see one commit)
```
