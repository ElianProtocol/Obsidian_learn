## Git — Объединение коммитов

### Цель

Узнайте, как объединять коммиты (squash commits).

### Инструкции

1. В репозитории Git создайте новый файл со содержимым "Mario" и создайте новый коммит.
2. Внесите изменения в только что созданный файл, изменив его содержимое на "Mario & Luigi", и создайте ещё один коммит.
3. Убедитесь, что у вас два отдельных коммита.
4. Объедините два последних коммита в один.

## Git - Объединение коммитов - Решение

1. В репозитории Git создайте новый файл со списком "Mario" и зафиксируйте изменения:

```
echo "Mario" > new_file
git add new_file
git commit -m "New file"
```

2. Внесите изменения в содержимое только что созданного файла, заменив его на "Mario & Luigi", затем создайте ещё один коммит:

```
echo "Mario & Luigi" > new_file
git commit -a -m "Added Luigi"
```

3. Убедитесь, что у вас два отдельных коммита, выполнив команду:

```
git log
```

4. Объедините два созданных вами коммита в один:

```
git rebase -i HEAD~2
```

Вы должны увидеть что-то похожее на:

```
pick 5412076 New file
pick 4016808 Added Luigi
```

Изменить `pick`на `squash`:

```
pick 5412076 New file
squash 4016808 Added Luigi
```

Сохраните его и добавьте сообщение для объединенного коммита.

> **Примечание** : Если при выполнении команды `git rebase -i HEAD~2`возникает фатальная ошибка (например, "invalid upstream 'HEAD~2'"), это обычно означает, что ваш второй коммит на самом деле является корневым коммитом, и перед ним нет допустимого родительского коммита. В этом случае вы можете либо:
> 
> - Используйте это `git rebase -i --root`для разрешения перезаписи корневого коммита, **или**
> - Создайте начальный коммит перед этими двумя коммитами, чтобы он `HEAD~2`указывал на действительные коммиты.

### После выполнения упражнения

**Ответьте на следующие вопросы:**

- **В чём причина объединения коммитов?**  
    История становится чище, и отслеживать изменения проще, без множества мелких коммитов, например, "удален символ".
    
- **Можно ли объединить более двух коммитов?**  
    Да.
