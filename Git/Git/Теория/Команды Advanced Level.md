==**Навигация и Слияние**==

	git checkout - переключение между ветками
		DevOps tip: Современная замена - git switch
	git merge - объединение двух веток. Создает специальный "мердж-коммит".
	git clone - копирование удаленного репозитория на твой ПК.
	git fetch - скачивает данные из удаленного репозитория, но **не вливает** их в твой рабочий код.
		Зачем? Чтобы безопасно узнать, что нового сделали коллеги, не рискуя сломать свои текущие изменения.
			DevOps tip: Всегда делай fetch перед rebase, чтобы твоя "база" была самой актуальной.

==Манипуляции с историей (База для DevOps)==

	git rebase - берет твои коммиты и "переносит" их в конец целевой ветки.
		Зачем? Чтобы история выглядела как прямая линия, а не как запутанная паутина.
	git revert - создает новый коммит, который отменяет изменения старого.
		Важно: Это "безопасный" способ откатить баг на сервере, не ломая историю
	git diff - показывает разницу между файлами.
		Пример: git diff main feature покажет, что именно изменил разработчик
