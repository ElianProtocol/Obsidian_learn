В DevOps идеология GitOps и Infrastructure as Code (IaC) - это фундамент. Без Git ты не сможешь управлять конфигурациями серверов или пайплайнами.

Концепции и База:
	*Git* - распределенная система управления версиями
	*Markdown* (.md) - твой главный инструмент для документации 
	*Ветки(branches)* - изоляция фич/исправлений
	*Теги(tags)* - "замороженные" точки в истории (релизы v1.0, v2.0)

| **Команда**           | **Описание**                       | **Зачем DevOps'у?**                      |
| --------------------- | ---------------------------------- | ---------------------------------------- |
| `git init`            | Создать новый репозиторий          | Начало нового проекта/инфраструктуры.    |
| `git add .`           | Подготовить файлы к сохранению     | Отбор изменений в конфигах или скриптах. |
| `git commit -m "..."` | Зафиксировать изменения            | Сохранение стабильной точки состояния.   |
| `git push`            | Отправить в облако (GitHub/GitLab) | Доставка кода на серверы или коллегам.   |
| `git pull`            | Забрать изменения                  | Обновление локальных скриптов.           |

 [[Стратегии ветвления (Branching Strategies)]]- это правила, по которым ведётся разработка в Git, там определяется, какие ветки всегда будут жить, как называть переменные, а также как и когда можно делать слияние веток.
